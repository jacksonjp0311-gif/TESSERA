# Tessera Repository Geometry

```text
RHP truth
  -> Nexus routes
    -> Geometry graph
      -> Lesson gates
        -> Agent CLI validation
          -> Tessera runtime
            -> Evidence
              -> bounded claims
```

The graph connects repository truth, routes, documentation, runtime,
validators, evidence, lessons, and claim boundaries.

The ordered loop is validated without deduplicating repeated stages. This
avoids the Hermes failure mode where a repeated stage could be silently
collapsed before order validation.

Non-claim lock: interconnection geometry is a repository contract, not proof
of correctness, capability, safety, production readiness, or transfer.
