# Tessera README / Nexus Discipline

## Purpose

This document imports repository-surface discipline from ASF-R, CMS, and RCC-N into Tessera.

Tessera needs more than a runnable engine. It needs a readable, navigable, validation-bound public surface so humans and AI agents can understand what the engine does, what it does not prove, where evidence lives, and what must be checked before any mutation or push.

## Imported Patterns

### ASF-R style loop visibility

Tessera must expose the whole governed loop:

```text
input -> orientation -> gates -> evidence -> wound if blocked -> certificate -> claim ceiling
```

The README must state what the loop does and what it does not authorize.

### CMS style current-state discipline

Tessera must maintain:

- current health snapshot,
- current public metrics,
- version lineage,
- README audit result,
- surface-alignment rule,
- public claim boundary.

### RCC-N style Nexus routing

Tessera must preserve:

- Human README,
- RCC Nexus README,
- AI Agent README,
- route map,
- context indexes,
- mini READMEs,
- Echo Location blocks,
- validation commands,
- claim ceilings.

## README Audit Law

```text
No public checkpoint promotion unless README, RCC route map, context indexes, AGENTS contract, public metrics, validation commands, and non-claim locks agree.
```

## Non-Claim Boundary

README discipline improves orientation. It does not prove code correctness, empirical validation, transfer, safety, production readiness, or AI understanding.
