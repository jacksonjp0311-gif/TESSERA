# TESSERA Engine v0.1 Architecture

## Core extraction

TESSERA Engine v0.1 turns the TESSERA theory line into a runnable neural reference engine:

```text
Input -> SparseSpectralField -> Code -> RateDistortion -> TriadicGates -> Replay -> Certificate -> Wound/Memory
```

## Canonical lock

```text
Memory is not activation.
Memory is certified compression.
Repair is not live until shadow-tested and authorized.
Synthetic success is not transfer.
```

## Runtime chain

```text
configure -> manifest -> split -> graph -> train codec -> evaluate -> gate -> replay -> compare baselines -> emit certificate
```

## Validity condition

A valid run must emit:

- dataset manifest
- graph declaration
- codec declaration
- rate-distortion metrics
- gate metrics
- replay/memory metrics
- wound ledger
- baseline report
- evidence package
- transfer certificate

## Falsification surface

TESSERA Engine v0.1 is weakened if:

- baselines outperform it under matched budget;
- false memory increases after a repair;
- final-test labels tune thresholds;
- low loss is presented as truth;
- synthetic success is presented as transfer;
- evidence artifacts are missing.
